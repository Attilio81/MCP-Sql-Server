# Comandi Git per Pubblicare su GitHub

## IMPORTANTE: Verifica Prima di Committare!
# Assicurati che questi file NON appaiano in git status:
# - .env
# - .claude/mcp.json
# - claude_desktop_config_TRANSITO.json
# - Qualsiasi file con password/credenziali

## Step 1: Verifica File
cd "C:\Progetti Pilota\MCPSqlServer"
git status

## Step 2: Inizializza Git (se necessario)
git init
git remote add origin https://github.com/Attilio81/MCP-Sql-Server.git

## Step 3: Aggiungi File
git add .

## Step 4: Verifica Cosa Stai per Committare
git status
# Controlla che NON ci siano file con credenziali!

## Step 5: Commit
git commit -m "Initial commit: MCP SQL Server v0.1.0

- Secure MCP server for SQL Server database inspection
- Connection pooling with configurable pool size
- Advanced security: SQL injection prevention, blacklist/whitelist
- Complete MCP tools: list_tables, describe_table, execute_query, get_table_relationships
- Comprehensive documentation and setup scripts
- Full test suite with connection validation

Features:
- Thread-safe connection pool with automatic recovery
- Multi-layer security validation
- Table blacklist with wildcard support
- Schema whitelist for access control
- Query validation (SELECT-only)
- Dangerous keyword detection
- Complete error handling with logging
- Production-ready with best practices

Documentation:
- Comprehensive README with examples
- Security best practices guide
- Contributing guidelines
- Claude Code integration guide
- Automated setup scripts

Tested with:
- Python 3.13
- SQL Server 2019
- ODBC Driver 17
- Claude Desktop & Claude Code"

## Step 6: Push
git branch -M main
git push -u origin main

## Step 7: Dopo il Push
# Vai su https://github.com/Attilio81/MCP-Sql-Server
# E verifica che:
# - README.md sia visualizzato
# - Non ci siano file .env o con credenziali
# - I badge siano visibili
# - LICENSE sia presente

## Step 8: Crea Release (Opzionale)
# Su GitHub:
# 1. Vai su "Releases"
# 2. Click "Create a new release"
# 3. Tag: v0.1.0
# 4. Title: v0.1.0 - Initial Release
# 5. Description: Copia da PUBLISH_TO_GITHUB.md
# 6. Click "Publish release"

## Comandi Utili

# Verifica remote
git remote -v

# Verifica branch
git branch -a

# Vedi ultimo commit
git log -1

# Vedi file tracciati
git ls-files

# Rimuovi file dal tracking (se necessario)
git rm --cached .env
git rm --cached .claude/mcp.json

# Annulla ultimo commit (se serve)
git reset --soft HEAD~1

# Forza push (ATTENZIONE! Solo se necessario)
git push -f origin main
